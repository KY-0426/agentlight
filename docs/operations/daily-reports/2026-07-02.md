# Agent Light 开发日报 · 2026-07-02

## 今日概要

完成桌面端云端同步链路重构（用量上报下沉至 Rust 后台线程）、云同步开关持久化、Docker 启动自动 migrate，并统一文档与 UI 中的 MySQL 开发指引。

## 完成事项

### 1. 桌面端云同步架构调整

- **用量上报从前端迁至 Rust**：移除 `App.tsx` 中的 `syncAgentUsage` 及 `tauriClient.uploadCodexThreadUsage`，统一由 `src-tauri/src/sync.rs` 后台线程轮询 Codex/Cursor 并排空队列。
- **Token 续期下沉 Tauri**：新增 `refresh_cloud_session` / `get_cloud_sync_enabled` 命令；前端 `ensureCloudAccessToken` / `refreshCloudAccessToken` 改为 invoke Rust，避免双端重复刷新。
- **刷新互斥锁**：`refresh_session_locked` 防止并发 refresh 竞态。

### 2. 云同步开关与登出清理

- 新增 `cloud-config.json` 持久化 `cloud_sync_enabled`（默认 true）。
- 登录保存 session 时自动开启同步；登出清除 session 时关闭同步并清空 `usage-queue.json`。
- 前端启动时通过 `getCloudSyncEnabled()` 同步 UI 状态。

### 3. 同步 worker 健壮性

- Debug 构建默认关闭用量同步，需设置 `AGENT_LIGHT_ENABLE_USAGE_SYNC=1` 才启用。
- 上报失败日志按状态码 5 分钟节流，避免刷屏。
- `model` 为 null 时不写入 JSON payload（修复服务端校验问题）。
- 新增 `usage_event_payload_omits_null_model` 单测。

### 4. 部署与文档（MySQL 统一）

- `server/Dockerfile`：多阶段构建、`docker-entrypoint.sh` 启动前自动 migrate、healthcheck `start-period=90s`。
- 删除 `scripts/setup-local-postgres.mjs` 及 `npm run db:setup`。
- 更新 `README.md`、`server/README.md`、`docs/engineering/*`、`docs/specs/task.md`：PostgreSQL → MySQL / CynosDB。
- 设置面板本地开发指引改为 `docker compose up -d mysql` + `npm run db:migrate`。

### 5. UI 小修

- `AiToolConnectModal`：仅当前安装中的工具显示「处理中…」，其它工具按钮不禁用文案误判。
